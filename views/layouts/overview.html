Layout

A layout is a template consisting of structural components intended to deliver consistency and cohesiveness among pages in an application. It's commonly defined in terms of spacing, grid systems and sections and contains components such as application bars, navigation and drawers.

Design Principles

The layouts provided reflect the following core design principals that are key to a quality experience. Before creating or selecting a layout, we encourage you to review these guidelines and adopt them as well.

  • Have a clear purpose - Understand what your user is trying to acheive and deliver the information they require.
  • Group related infomation - Organizing the information according to relationships makes it more efficient for users to work
  • Create a visual hierarchy - give the most important information the greatest prominence.
  • Provide context - Keep people aware of where they are. Ensure that the design is self-contained and avoid breaking people out of the experience
  • Simplify and Clarify - Reduce visual "noise". Create layouts that are lighter, faster and easy to read. Avoid distractions, jargon, and long loading times.
  • Be consistent - Choose layouts that are predictable and familiar, this creates an experience that is recognizable across products.
Configurations

A page layout can take on a number of different configurations, based on the use cases an application aims to support. Provided here are some of the most common layout structures, but you are free to create the layout that works best for you.

Use the layout classes below to create as simple or robust a layout as you need. For guidance on arranging elements follow the layout principles in creating layouts that make the most sense for your application.

Classes

This table gives you a quick overview of the CSS classes that can be applied to create various layouts.

Class Description
.app Applied to the <html> tag to create a full-screen application layout
.hbox Horizontal wrapper that can contain columns. To create columns add .aside. Atternately used with the bootstrap grid system.
.stretch Sets the .hbox to 100% height in an application layout
.aside-* Defines a column inside an .hbox. Set the width of the aside using .aside-*. Available sizes are -xxs, -xs, -sm, -md, -lg, -xl, and -xxl.
.vbox Vertical wrapper that can contain a <header>, scrollable <section> and <footer> in an application layout.
.flex Allows for a flexible height of the <header> the <footer> in .vbox layout
.header Sets the height of the <header> tag in a .vbox layout. A .header-md can be applied as well to increase the height of the <header>
.scrollable Applied to the <section> tag in a .vbox layout. Makes the main content area scrollable
.w-f Applied to the <section> tag in a .vbox layout. Reduces the height of the main content allowing when a <footer> is applied.
.footer Sets the height of the <footer> tag in a .vbox layout. A .header-md can be applied as well to increase the height of the <header>
Layout Types

A type refers to either the function of a page — for example, a Home page —or its layout, such as master-detail.

Form Layout

Forms are the primary place where users interact with an application to complete tasks; they allow a user to enter and read data. For example, forms can be used to enter billing or product information when creating an order. Forms can contain Text Fields, Text Areas, Select Dropdowns, Radio Buttons, Checkboxes, File Uploads, Date Pickers, and other methods of input.

Master-Detail Layout

A master-detail groups together two layout views in a one-to-many relationship. The information on the entire page displays one or more master objects and displays a detail object based on selection of a master. This is often expressed in a Split View consisting of two side-by-side panels: an item list and the selected item’s details but may also include a drilldown or a popup.

Table Layout

A table is the most basic format for displaying a list. Each record is represented by a single row of data that begins with the record’s primary field and shows additional fields in subsequent columns. The data is labeled using column headers that can be interactive.
