Form Layout

Forms are the primary place where users interact with an application to complete tasks; they allow a user to enter and read data. For example, forms can be used to enter billing or product information when creating an order. Forms can contain Text Fields, Text Areas, Select Dropdowns, Radio Buttons, Checkboxes, File Uploads, Date Pickers, and other methods of input.

There are two types of forms and each has a designated purpose.

  • Transactional forms are used for both required and optional input and are intended to present all editable attributes of a single object. They are generaly used when creating or modifying objects like invoices, expenses, payments, etc.
  • Sequential Forms divide simple or complex tasks into a series of steps, with the goal of reducing the probability of user error and may be used to edit or create multiple objects with interdependencies.
